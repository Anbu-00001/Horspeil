# Hörspiel — Google Stitch Prompt Pack (Phase 1)

> Curated, paste-ready prompts for **Google Stitch** to generate the Phase 1 screens of
> **Hörspiel** (German-only podcast platform; Spotify-style upload/listen + YouTube-style
> social, gated so only German audio is accepted). Flutter, Material 3, mobile-first.
> Generated by the design-prompt agent on 2026-07-13.

## How these prompts were written (Stitch best practices applied)

- **Specificity over vagueness** — named screens, explicit components, states, layout hierarchy.
- **Exact hex codes, never color names** — `#C8621C` returns the same shade every time; "amber" drifts.
- **Zoom-out → zoom-in shape** — restate product context, then the screen goal, then components/states.
- **Design system first, screens second** — Prompt A sets tokens once; every screen references it.
- **~80–160 words per screen prompt** — complete but not so long Stitch loses the thread.
- **One change per refinement** — stacking edits makes Stitch rebuild the whole layout.

## A. Global design-system prompt (paste this FIRST)

```text
You are designing Hörspiel, a mobile-first German-only podcast app built in Flutter
(Material 3) for Android and iOS — think Spotify-style audio upload and listening plus
YouTube-style social affordances, but every podcast is spoken in German. Brand
personality: warm, literary, audio-drama ("Hörspiel" = radio play). Establish a reusable
design system applied to every screen.

Color palette (use these exact hex values):
- Paper/background #FAF6F0, surface #FFFFFF, hairline/border #E7DFD3
- Primary amber "Bernstein" #C8621C, pressed #A64E12
- Secondary forest green #2F5D50, success #2E7D5B, error/brick #B23A2E
- Text ink #1F1B16, muted text #6B6157
- Dark theme: background #16130F, surface #211C16, text #F3ECE1, amber #E0842F

Typography: headings in a warm literary serif (Fraunces or Lora); body, labels and
buttons in a clean humanist sans (Inter). Components: rounded 16–20px cards, pill-shaped
category chips, a large circular record FAB, soft warm shadows, generous spacing. Bottom
tab nav: Startseite, Entdecken, Aufnehmen (center amber FAB), Profil. Support light and
dark. Tone: warm, encouraging, German microcopy. Do not imitate Spotify or YouTube branding.
```

## B. One prompt per Phase 1 screen

### 1. Onboarding / Auth

```text
Design the Hörspiel onboarding and authentication screens using the established design
system (amber #C8621C on warm paper #FAF6F0, serif headings, Inter body, Material 3).
Screen one: a warm welcome with the Hörspiel wordmark, a headline "Podcasts. Nur auf
Deutsch." and one line explaining the German-only hook, plus a large primary button
"Kostenlos starten" (Registrieren) and a text link "Anmelden". Screen two: an email
sign-up / login form with E-Mail and Passwort fields, a "Passwort vergessen?" link, a
primary amber "Weiter" button, and optional social buttons (Google, Apple) below a subtle
divider labelled "oder". Add a small audio-wave motif behind the wordmark. Show input
focus and inline error states. Keep it clean, inviting and literary, mobile-first.
```

### 2. Home feed (Startseite)

```text
Design the Hörspiel home feed (Startseite) using the design system. Top: greeting
"Willkommen zurück" with a small avatar and a search icon. Below, a horizontally
scrollable row of pill category chips (Essen, Reisen, Kultur, Blogs, Bildung, Geschichte);
the selected chip is filled amber #C8621C. Main area: one larger "featured" hero card,
then a vertically scrollable list of German podcast cards — each with rounded cover art,
episode title (serif), creator name, category tag, duration, and a small circular play
button. Bottom navigation bar: Startseite, Entdecken, Aufnehmen (center amber record FAB),
Profil. Warm paper background, soft shadows, generous spacing. Include a subtle loading
skeleton state for the cards. All copy in German, Material 3, mobile-first.
```

### 3. Category browse / filter (Kategorien)

```text
Design the Hörspiel category browse and filter screen (Kategorien) using the design system.
Header "Entdecken" with a search field. A grid of category tiles — Essen, Reisen, Kultur,
Blogs, Bildung, Geschichte, Musik, Wissenschaft — each a rounded tile with an icon and
label on a soft amber/forest-tinted background. On selecting a category, show a filtered
results state: the active category as a filled amber #C8621C chip with an "x" to clear, a
result count ("128 Hörspiele"), and a sort control ("Neueste"). Results reuse the same
podcast list cards as the home feed for visual consistency. Include a friendly empty state
("Noch keine Hörspiele hier"). Warm paper background, Inter labels, serif section titles,
bottom tab nav. Material 3, mobile-first.
```

### 4. Record screen (Aufnehmen)

```text
Design the Hörspiel in-app recording screen (Aufnehmen) using the design system, in a calm
dark-studio theme (background #16130F). Center a big circular amber #C8621C record button;
while recording it pulses and shows a stop icon. Above it, a large monospaced timer (00:00)
and a live animated waveform in amber. Below, secondary controls: "Pause" and "Erneut
aufnehmen" (re-record). Top bar with a close "X" and the title "Neue Aufnahme". After
stopping, show a review state: the captured waveform, a playback scrubber, take duration,
and a prominent primary button "Weiter" leading to upload. Keep the layout focused and
uncluttered. Serif heading, Inter labels, generous spacing, German microcopy throughout.
Material 3, mobile-first.
```

### 5. Upload + language-gate flow (the signature screen)

```text
Design the Hörspiel upload and language-gate flow using the design system — this is the
app's signature moment, so show all three connected states.
State 1 "Wird geprüft": after picking or recording audio, a calm centered screen
"Sprache wird geprüft…" with an animated amber waveform-into-checkmark loader and
reassuring copy ("Wir hören kurz rein, ob dein Hörspiel auf Deutsch ist.").
State 2 ANGENOMMEN: a success screen with a forest-green #2E7D5B badge "Deutsch erkannt",
then a publish form — Titel, Kategorie (chips/dropdown), Beschreibung, Titelbild upload —
and a primary amber "Veröffentlichen" button.
State 3 ABGELEHNT: a friendly, non-punishing screen with a brick #B23A2E badge "Kein
Deutsch erkannt", a short empathetic explanation, and two actions: "Erneut versuchen" and
a secondary "Manuelle Prüfung anfordern".
Warm, human tone; no harsh error styling. Material 3, serif headings, Inter body, mobile-first.
```

### 6. Player / playback (Wiedergabe)

```text
Design the Hörspiel now-playing / player screen (Wiedergabe) using the design system,
full-screen on warm paper (also provide the dark #16130F variant). Large rounded cover art
centered; below it the episode title in serif, the creator name, and a category tag pill in
amber #C8621C. A scrubber/progress bar with elapsed and remaining time. Transport controls
centered on a big amber play/pause button, flanked by 15-second skip-back, skip-forward,
and a playback-speed control. Top bar: a down-chevron to minimize and a share icon. Include
muted, clearly-secondary placeholder affordances for like (heart) and comment (speech
bubble) — visible but inactive, since social features come later. A slim "Als Nächstes"
hint at the bottom. Immersive, warm, literary. Material 3, Inter labels, generous spacing.
```

### 7. Profile (minimal) (Profil)

```text
Design the Hörspiel profile screen (Profil) using the design system. Header: a circular
avatar, display name in serif, an @handle, a one-line bio, and small stats (Hörspiele,
Follower, Folgt). A secondary "Profil bearbeiten" button and a settings gear icon top-right.
Below, a section titled "Meine Hörspiele" listing the user's uploaded podcasts as compact
cards — cover thumbnail, title, category tag, duration, play count, and a small overflow
menu. A minimal segmented control ("Hochgeladen" / "Gespeichert"). Include an empty state
for new users: "Noch keine Hörspiele" with a primary amber "Erste Aufnahme starten" button
linking to Aufnehmen. Warm paper background, bottom tab nav with Profil active. Material 3,
serif headings, Inter body, German throughout.
```

## C. How to use these in Stitch

1. **Paste order** — paste Prompt A (design system) first, then generate each screen from B
   one at a time, in its own generation (cleaner, more coherent output than all-at-once).
2. **Standard vs Experimental mode** — Standard (Gemini 2.5 Flash) for these text prompts;
   switch to Experimental (Gemini 2.5 Pro) for higher fidelity or to feed a reference image
   (e.g. a sketch of the record waveform or the language-gate result). Experimental has a
   lower monthly limit — use it selectively.
3. **Iterate one change at a time** — refine with single-change follow-ups; stacking edits
   makes Stitch rebuild and break what already worked.
4. **Keep colors locked to hex** — always restate hex when refining; never "the orange".
5. **Enforce cross-screen consistency** — Shift-select multiple screens and paste one theme
   prompt to unify; reuse the exact same podcast-card description across Home/Category/Profile.
6. **Signature screen gets extra passes** — spend the best iterations on the language-gate
   result (Prompt 5): the "Deutsch erkannt" vs "Kein Deutsch erkannt" tone is the differentiator.
7. **Export** — export to Figma or copy the front-end code to hand off toward Flutter/Material 3.
